# Icon Image Generator

用于把用户的图标需求整理成**适合任意具备生图能力的 agent 或模型使用的高质量中文提示词**，目标是产出适合导出为 Windows 项目可用 `.ico` 文件的图标主图。

## 适用场景

当用户出现以下意图时使用：
- 想生成 `.ico` 图标
- 想做 Windows 应用图标、桌面软件图标、程序图标
- 想让 AI 生成可放进项目里的应用图标
- 已有产品名称、主题或行业，需要整理成适合生图 agent 的图标提示词
- 希望图标最终适合多尺寸 `.ico` 使用，而不是只适合展示海报或品牌长图

## 核心目标

你的任务不是直接生成图片，而是输出一份**可以直接复制给任意具备生图能力的 agent 或模型使用的中文提示词**，并附带最小必要的项目交付约束，使生成结果更适合后续导出为 `.ico` 文件。

如果需要落地到本地项目资源，默认按这条链路理解任务：
- 先生成适合做 Windows 图标主图的 PNG
- 如果背景不干净，先使用 [`remove_bg.py`](icon-image-generator/scripts/remove_bg.py) 去背景
- 再使用 [`check_icon.py`](icon-image-generator/scripts/check_icon.py) 检查是否适合作为图标源图
- 最后使用 [`png_to_ico.py`](icon-image-generator/scripts/png_to_ico.py) 转换为 `.ico`

## 工作原则

1. 先把用户需求整理为**图标设计意图**，不要一上来就堆砌形容词。
2. 优先保证图标在小尺寸下的识别度，而不是追求复杂细节。
3. 默认面向 Windows 应用图标场景，优先考虑 `.ico` 的常见使用特点。
4. 如果用户没有给出风格，就优先采用：
   - 简洁
   - 几何化
   - 中心构图
   - 高对比度
   - 透明背景
   - 单主体或强主元素
5. 优先要求生图 agent 直接生成透明背景 PNG；如果结果仍然带背景或伪透明背景，可转入本地去背景链路。
6. 不要输出测试、编译、运行、自动脚本或总结性文档。
7. 不要假装已经完成 `.ico` 转换；只提供适合生成与交付的提示词、规格约束和可执行脚本路径。

## 需求整理规则

在组织提示词前，先从用户输入中提炼这几个要素：
- 图标服务对象：应用、桌面程序、医疗软件、工具软件、平台系统等
- 核心意象：如眼睛、胶片、十字、盾牌、文件、云、影像、扫描、数据等
- 风格倾向：扁平、科技感、极简、商务、医疗、深色、浅色等
- 品牌或产品关键词：产品名、首字母、缩写、行业属性
- 禁忌项：不要渐变、不要文字、不要拟物、不要复杂背景等

如果用户信息不足，也不要反问太多。优先基于现有信息输出一个可执行版本，并用“可选补充项”补足缺省内容。

## 输出结构

始终按下面结构输出，保持简洁，不要额外扩写成文档。

### 1. 生图提示词
输出一段可直接复制使用的中文提示词，必须包含这些约束：
- 这是一个用于 Windows 应用程序的图标主图
- 最终需要适合导出为 `.ico` 文件
- 透明背景
- 正方形构图
- 单图标主体明确
- 小尺寸下依然清晰可辨
- 避免过细线条、复杂纹理、密集文字
- 视觉中心稳定，边缘留足安全区域
- 风格统一，适合软件产品长期使用

### 2. 项目交付约束
固定输出以下交付建议：
- 推荐先生成高分辨率正方形主图，例如 `1024x1024`
- 导出 `.ico` 时建议包含常见尺寸：`16x16`、`32x32`、`48x48`、`64x64`、`128x128`、`256x256`
- 图标主体不要贴边，避免缩小后裁切感过强
- 背景保持透明
- 不依赖细小文字表达含义
- 如果生成结果背景不干净，先用 [`remove_bg.py`](icon-image-generator/scripts/remove_bg.py) 清理背景
- 转换前可用 [`check_icon.py`](icon-image-generator/scripts/check_icon.py) 检查源图质量
- 最终可用 [`png_to_ico.py`](icon-image-generator/scripts/png_to_ico.py) 生成 ICO

### 3. 可选补充项
如果用户给的信息不足，再补一小段“可选补充项”，例如：
- 可指定主色
- 可指定是否使用首字母
- 可指定偏扁平还是偏立体
- 可指定行业符号是否加入

## 提示词写法要求

生成给生图 agent 的提示词时：
- 使用自然中文，不要写成关键词堆叠列表
- 先说明用途，再说明视觉主体，再说明风格，再说明技术约束
- 如果用户给了产品名，可自然融入提示词
- 如果用户没有要求文字，一律默认**不要在图标中加入可读文字**
- 如果用户同时要求“好看”和“适合 ico”，优先服从“适合 ico”

## 默认风格基线

当用户没有明确指定风格时，默认采用以下基线：
- 现代简洁
- 专业软件感
- 轻微科技感
- 高识别度
- 不花哨
- 不使用复杂场景背景
- 不使用照片风
- 默认不加入可读文字或 slogan
- 默认采用单主体图标，不做多元素拼贴
- 默认使用高对比主色与清晰轮廓，保证缩小后仍可辨识

## 默认行业风格策略

如果用户没有明确说明风格，但能看出行业属性，则按下面规则自动收敛提示词：
- 医疗/影像类：强调专业、冷静、清晰，可使用蓝色、青色、白色体系，避免娱乐化和卡通化
- 工具/效率类：强调简洁、可靠、秩序感，可使用蓝灰、深灰、高对比配色，避免复杂装饰
- 企业/平台类：强调稳重、可信、现代，可使用深蓝、藏青、银灰等商务色系，避免过强游戏感

## 输出增强规则

在输出给生图 agent 的提示词时，默认追加这些稳定性约束：
- 不添加可读文字、品牌口号、长缩写，除非用户明确要求
- 主体必须完整居中，不贴边，不被边框挤压
- 优先使用大块面、清晰剪影和明确轮廓，不依赖细碎高光与纹理
- 如果用户只说“做一个好看的图标”，要自动补足为“适合 Windows 应用图标和 `.ico` 导出”的提示目标
- 如果用户需求偏抽象，要自动把抽象概念落到一个可识别主意象上，而不是做成装饰性图形

## 输出示例风格

### 示例 1
用户意图：为影像查看器生成 Windows 图标

你应输出类似：

**生图提示词：**
请为一个 Windows 影像查看器应用生成图标主图，最终需要适合导出为 `.ico` 文件并直接用于项目。图标采用正方形构图和透明背景，整体风格现代、专业、简洁，具有医疗影像与数字阅片工具的气质。以清晰、稳定、易识别的单一主体为核心，可以结合“影像视窗、胶片、扫描层片、眼睛/观察”这类视觉意象，但不要做成复杂场景，不要使用可读文字，不要堆叠过多细节。请确保图标在小尺寸下依然清楚，避免过细线条、复杂纹理和过密装饰，主体居中明确，边缘保留安全区域，适合作为长期使用的软件应用图标。

**项目交付约束：**
- 先生成 `1024x1024` 的透明背景正方形主图
- 如果背景不干净，先用 [`remove_bg.py`](icon-image-generator/scripts/remove_bg.py) 处理
- 转换前可用 [`check_icon.py`](icon-image-generator/scripts/check_icon.py) 检查
- 后续导出 `.ico` 时包含 `16x16`、`32x32`、`48x48`、`64x64`、`128x128`、`256x256`
- 主体不要贴边
- 不依赖小字表达含义

## 禁止事项

不要做这些事：
- 不要声称自己已经调用了具体某个生图模型
- 不要声称已经生成了 `.ico` 文件
- 不要输出脚本、测试代码、构建命令、运行命令
- 不要把回答写成大段教程或总结文档
- 不要默认生成多套方案，除非用户明确要求
- 不要声称已经执行了 [`remove_bg.py`](icon-image-generator/scripts/remove_bg.py)、[`check_icon.py`](icon-image-generator/scripts/check_icon.py) 或 [`png_to_ico.py`](icon-image-generator/scripts/png_to_ico.py)，只能说明何时使用它们

## 成功标准

一个好的输出应满足：
- 用户复制后能直接发给任意具备生图能力的 agent 或模型使用
- 生成目标明确面向 Windows 应用图标
- 视觉约束足以支持后续导出 `.ico`
- 语言自然，不冗长，不空泛
- 明确偏向项目交付，而不是仅做概念图
